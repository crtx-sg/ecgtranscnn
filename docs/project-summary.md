# ECG-TransCovNet: Project Technical Summary

## 1. What It Is

ECG-TransCovNet is an AI system that classifies **16 cardiac arrhythmia conditions** from 7-lead ECG signals in real time. It is designed for patient monitoring scenarios where ECG alarms arrive as 12-second signal windows and need automated classification.

The system covers the full pipeline from signal generation to real-time inference:

```
Synthetic ECG Simulator  -->  Training Pipeline  -->  Trained Model
                                                          |
Real-time HDF5 files  -->  Preprocessing  -->  Model  -->  Classification Report
```

### Conditions Detected

| Category | Conditions |
|----------|-----------|
| Normal variants | Normal Sinus Rhythm, Sinus Bradycardia, Sinus Tachycardia |
| Supraventricular | Atrial Fibrillation, Atrial Flutter, PAC, SVT |
| Ventricular | PVC, Ventricular Tachycardia, Ventricular Fibrillation |
| Conduction | LBBB, RBBB, AV Block 1st, 2nd Type I, 2nd Type II |
| Ischemic | ST Elevation |

## 2. Architecture

The model is a **hybrid CNN-Transformer** based on Shah et al. (IET CIT 2024), combining convolutional feature extraction with transformer attention for global context.

```
7-lead ECG (7 x 2400 samples, 12s at 200 Hz)
  |
  v
CNN Backbone (Selective Kernel Conv + Residual Blocks)
  7 x 2400  -->  128 x 38  (local features)
  |
  v
Transformer Encoder (3 layers, 8 heads)
  128 x 38  -->  38 x 128  (global temporal context)
  |
  v
Transformer Decoder (3 layers, 8 heads)
  16 learnable object queries (DETR-style, one per condition)
  |
  v
Classification Head (FFN per query: 128 -> 64 -> 1)
  |
  v
16-class output
```

**Key design choices:**
- **Selective Kernel Convolution** adapts receptive field size per input via multi-branch attention (kernels 3, 5, 7)
- **DETR-style object queries** let each decoder query specialize to one cardiac condition via cross-attention
- **Focal Loss** (gamma=2.0) with inverse-frequency class weights addresses condition prevalence imbalance

## 3. ECG Simulator

All training and evaluation data is generated by a built-in **synthetic ECG simulator** -- no real patient data is required. The simulator produces physiologically realistic signals with:

- **Beat morphology**: P wave, QRS complex, T wave with condition-specific variations (e.g., absent P waves in AFib, wide QRS in bundle branch blocks, prolonged PR in AV blocks)
- **7-lead system**: ECG1-3, aVR, aVL, aVF, vVX with inter-lead correlations from the Einthoven triangle
- **Supplementary signals**: PPG (75 Hz), respiratory waveform (33 Hz), vital signs (HR, SpO2, BP, RR, Temp)
- **HDF5 output**: Phase-0 schema compatible files with metadata, signals, vitals, and ground truth labels

### Noise Pipeline

The simulator applies six types of noise artifacts, each configurable:

| Artifact | Frequency Range | Source |
|----------|----------------|--------|
| Baseline wander | 0.1--0.5 Hz | Patient breathing, movement |
| Gaussian noise | Broadband | Electronic instrumentation |
| EMG bursts | 20--90 Hz | Muscle activity |
| Motion artifacts | Transient spikes | Electrode displacement |
| Powerline interference | 50/60 Hz | Electrical equipment |
| Electrode contact loss | Intermittent dropout | Poor electrode adhesion |

Four noise presets (clean, low, medium, high) plus a mixed mode for training robustness.

## 4. Signal Preprocessing (New)

A configurable **preprocessing pipeline** was added to clean ECG signals before classification. Previously, only per-lead z-score normalization was applied -- noise passed through to the model untouched.

### Filter Pipeline

Applied per-lead using zero-phase IIR filters (preserves QRS timing):

| Stage | Filter | Target |
|-------|--------|--------|
| 1 | Median filter (optional) | Motion artifact spikes |
| 2 | Butterworth high-pass (0.5 Hz) | Baseline wander |
| 3 | IIR notch (50 Hz) | Powerline -- Europe/Asia |
| 4 | IIR notch (60 Hz) | Powerline -- Americas |
| 5 | Butterworth low-pass (40 Hz) | EMG, high-freq noise |
| 6 | Z-score normalization | Amplitude variation |

### Presets

| Preset | Description |
|--------|------------|
| `none` | Z-score only (backward compatible, current default) |
| `default` | Standard clinical filtering -- recommended |
| `conservative` | Wider passband, minimal signal alteration |
| `aggressive` | Includes median spike removal, tighter passband |

### Filter Validation

| Test | Criterion | Result |
|------|-----------|--------|
| Baseline wander removal | >= 20 dB attenuation | Pass |
| 50 Hz powerline removal | >= 20 dB attenuation | Pass |
| 60 Hz powerline removal | >= 20 dB attenuation | Pass |
| QRS peak timing | Within +/-1 sample | Pass |
| Processing speed | < 50 ms per 7-lead signal | Pass (< 5 ms) |

## 5. Training Pipeline

- **Optimizer**: AdamW (lr=5e-4, weight decay=1e-4)
- **Schedule**: Linear warmup (5 epochs) + cosine annealing
- **Loss**: Focal Loss with per-class inverse-frequency weights
- **Augmentation**: Random time shift, amplitude scaling, Gaussian noise, channel dropout
- **Early stopping**: Patience 20 epochs on validation accuracy
- **Data caching**: Generated datasets cached to disk (keyed by noise level, distribution, filter preset)

---

## 6. Inference Pipeline

A **real-time directory-watching processor** monitors a folder for incoming HDF5 files (via Linux inotify), runs inference per event, and prints classification results with aggregate metrics.

```
HDF5 file arrives  -->  inotify detects  -->  Parse events  -->  Preprocess  -->  Model  -->  Print results
```

On shutdown (Ctrl+C), prints an aggregate classification report with per-condition precision, recall, F1, and macro F1.

## 7. Model Performance

### Baseline: Clean Data (No Noise)

Best model achieves **96.4% accuracy** on clean synthetic data (500 samples, balanced):

| Metric | Value |
|--------|-------|
| Accuracy | 96.4% |
| Macro F1 | 0.966 |
| Macro Precision | 0.968 |
| Macro Recall | 0.965 |

9 of 16 conditions achieve F1 >= 0.95 on clean data. Strongest: AFib, VFib, VTach, SVT, PVC (all near-perfect). Weakest: AV Block subtypes.

### Noise Degradation (Without Preprocessing)

| Noise Level | Accuracy | Macro F1 |
|-------------|----------|----------|
| Clean | 96.4% | 0.966 |
| Medium | 51.2% | 0.488 |
| High | 30.8% | 0.266 |

The model loses ~65 percentage points of accuracy going from clean to high-noise signals. Many conditions (Normal Sinus, Sinus Brady/Tachy, PAC, AV Blocks, ST Elevation) drop to near-zero recall under high noise.

### Preprocessing Impact

Adding the default filter preset **before** classification (no retraining, same model checkpoint):

| Noise Level | Without Filter | With Filter | Improvement |
|-------------|---------------|-------------|-------------|
| Medium -- Accuracy | 51.2% | 67.0% | **+15.8 pp** |
| Medium -- Macro F1 | 0.488 | 0.674 | **+0.186** |
| High -- Accuracy | 30.8% | 41.0% | **+10.2 pp** |
| High -- Macro F1 | 0.266 | 0.352 | **+0.086** |

Notable per-condition improvements on high noise (F1, unfiltered vs filtered):

| Condition | No Filter | Filtered | Change |
|-----------|-----------|----------|--------|
| LBBB | 0.615 | 0.889 | +0.274 |
| SVT | 0.750 | 0.837 | +0.087 |
| PVC | 0.312 | 0.588 | +0.276 |
| VTach | 0.857 | 0.897 | +0.040 |
| ST Elevation | 0.000 | 0.389 | +0.389 |

### Trained Model Inventory

| Checkpoint | Training Data | Accuracy (Clean) |
|------------|--------------|-------------------|
| `models/avblock_fix/best_model.pt` | 16K clean, AV block-focused | 96.4% |
| `models/improved/best_model.pt` | 16K clean, balanced | 90.0% |
| `models/noise_robust/best_model.pt` | 16K mixed-noise | 87.4% |

### Known Limitations

- **AV Block subtypes** remain challenging -- subtle PR interval differences are hard to distinguish, especially under noise
- **Normal Sinus vs. AV Block 1st Degree** confusion persists (F1 ~0.65)
- Model was trained on clean data -- retraining on filtered noisy data would likely yield further gains
- All evaluation is on synthetic data; real clinical data validation is pending
